SAP PO REST Adapter Module Configuration —— XML 与 JSON 转换配置指南

本文最后更新于 2026-06-26

引言

在 SAP PO（Process Orchestration）开发 REST 接口时，XML 与 JSON 之间的自动转换是高频需求。REST Adapter 默认的转换行为有时无法满足第三方系统的接口规范，最常见的场景就是 JSON Root Array 问题。

本文记录了我实际开发中遇到并验证的 setIgnoredElements 配置方案，以及通过资料查询整理的 RESTAdapterBean 常用 Module Configuration 参数速查。

⚠️ 可靠性说明： setIgnoredElements 参数已经过实际项目验证，确认有效。后续扩展参数来自 GPT 联网搜索结果，尚未逐一实测，使用前建议先在测试环境验证。

四、适用场景

此方案适用于各类需要 JSON Root Array 的第三方系统集成：

  • MES / WMS / SRM / OA
  • 微信小程序、企业微信
  • Java SpringBoot
  • .NET WebAPI
  • Python FastAPI

相比修改 ESR Message Type 或 Message Mapping，该方法更加简单优雅，且不影响 XML Schema 结构设计。

五、RESTAdapterBean 常用 Module Configuration 参数速查

REST Adapter 的 Module Key 为：

以下参数由资料查询整理（仅 setIgnoredElements 实际验证过）。

1. setIgnoredElements ⭐⭐⭐⭐⭐

忽略 XML 指定节点，去除包装层，输出 Root Array。

2. setIgnoreEmptyArrayValues ⭐⭐⭐⭐⭐

忽略空数组，精简 JSON 输出。

适用于第三方接口不允许空数组的场景。

3. ProceedRedirects ⭐⭐⭐⭐☆

控制是否自动跟随 HTTP 重定向（301/302/307）。

4. EncodeURL ⭐⭐⭐⭐☆

是否自动进行 URL Encode。

5. errorFormat

REST Sender 返回异常时的错误信息格式。

6. escapeNonXML10Symbols

转义非法 XML 1.0 字符（特殊字符、控制字符、非法 Unicode），避免 XML Parser 报错。

7. escapeRestricted

自动转义 XML 保留字符（& < >），避免 XML 解析失败。

8. addXMLDeclaration

控制是否输出 XML 声明。

9. setStandaloneNo

输出 standalone=”no” 的 XML 声明。

10. setRequestXIDynamicHeaders

控制 Dynamic Header 与 HTTP Header 之间的互相转换。适用于 OAuth Token、自定义 Header 等场景。

11. useDefaultCharsetForCustomHandling

指定默认字符集，常用于解决中文乱码（UTF-8、GBK）。

12. setIgnoreNamespaces

忽略 XML Namespace，适用于 JSON Key 不需要 Namespace 前缀的场景。

六、推荐组合配置（REST Receiver）

对于大多数 REST Receiver 接口，可参考以下组合：

Parameter 推荐值 说明
setIgnoredElements IT_TAB 去除包装节点，输出 Root Array
setIgnoreEmptyArrayValues true 空数组不输出
EncodeURL true URL 自动编码
ProceedRedirects true 自动处理 HTTP 重定向
useDefaultCharsetForCustomHandling true 避免字符集问题

七、总结

RESTAdapterBean 的 Module Configuration 是 SAP PO REST Adapter 中一个非常实用但容易被忽略的高级功能。

核心要点：

参数 推荐指数 用途
setIgnoredElements ⭐⭐⭐⭐⭐ 去除包装节点，输出 Root Array
setIgnoreEmptyArrayValues ⭐⭐⭐⭐⭐ 忽略空数组
EncodeURL ⭐⭐⭐⭐☆ URL 自动编码
ProceedRedirects ⭐⭐⭐⭐☆ 自动处理 HTTP 重定向
errorFormat ⭐⭐⭐⭐☆ 返回 JSON 错误信息

其中，setIgnoredElements 是开发 REST 接口时最值得掌握的配置，可在保持 ESR/XSD 结构不变的情况下，轻松满足第三方接口对 JSON 根数组格式的要求。
