5. 内表类型与性能

选择合适的内表类型对程序性能影响很大。

Standard Table（标准表）

  • 默认内表类型
  • 线性查找，复杂度 O(n)
  • 适合按索引访问或顺序处理

Sorted Table（排序表）

  • 自动按 KEY 排序
  • 二分查找，复杂度 O(log n)
  • 数据始终有序

Hashed Table（哈希表）

  • 哈希查找，复杂度 O(1)
  • 速度最快，但不支持索引访问
  • 必须指定 UNIQUE KEY

性能对比

建议： 如果内表主要用于按 KEY 读取数据，优先选择 HASHED TABLE 或 SORTED TABLE，避免 STANDARD TABLE + READ TABLE 的组合。

总结

本文梳理的语法技巧涵盖了 ABAP 日常开发中常见的场景，核心思想是：

  1. 用新语法替代传统写法 — 代码更简洁、可读性更强
  2. 选对数据结构 — 根据访问模式选择合适的表类型，O(1) 和 O(n) 在实际业务中差距显著
  3. 把计算推到数据库层 — 减少数据传输和应用服务器负担
  4. 注意边界情况 — 表表达式的异常处理、税率的系统差异、日期时区等细节
