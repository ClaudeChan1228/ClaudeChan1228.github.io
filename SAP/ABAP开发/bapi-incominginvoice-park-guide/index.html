使用 BAPI_INCOMINGINVOICE_PARK 创建预制采购发票

本文最后更新于 2026-06-26

引言

BAPI_INCOMINGINVOICE_PARK 是 SAP MM 模块中用于创建预制采购发票（Park Invoice） 的标准 BAPI。在采购入库（MIGO / BAPI_GOODSMVT_CREATE）完成后，程序调用该 BAPI 将采购发票预制到 SAP 中，从而形成 采购订单 → 入库 → 发票 的完整 MM 业务闭环。

本文基于实际接口开发经验，梳理了从组装发票行项目到调用 BAPI、事务提交流程的完整实现。

整体流程

1. 构造发票行项目 (ItemData)

入库成功后，程序循环采购订单明细，为 BAPI 组装发票行项目。

字段说明

字段 说明
REF_DOC 采购入库物料凭证号
REF_DOC_YEAR 物料凭证年度
REF_DOC_IT 物料凭证行号
PO_UNIT 采购订单单位
QUANTITY 开票数量
ITEM_AMOUNT 本行金额（未税）
TAX_CODE 税码

为什么需要 REF_DOC？

REF_DOC、REF_DOC_YEAR、REF_DOC_IT 三者共同指定了：

本次采购发票对应哪一张采购入库物料凭证。

3. 构造发票抬头 (HeaderData)

字段说明

字段 说明
DOC_DATE 凭证日期
PSTNG_DATE 过账日期
COMP_CODE 公司代码
CURRENCY 币种（来自采购订单）
INVOICE_IND 普通发票标识
DOC_TYPE 发票凭证类型 RE
CALC_TAX_IND 自动计算税额

INVOICE_IND = ‘X’

表示本次创建的是普通采购发票，而非：

  • Credit Memo（贷项凭证）
  • Subsequent Debit（后续借记）
  • Subsequent Credit（后续贷记）

DOC_TYPE = ‘RE’

SAP 标准的采购发票凭证类型。所有通过该 BAPI 生成的 MM 发票都使用 RE。

CALC_TAX_IND = ‘X’

告诉 SAP：

税额不由接口计算，由 SAP 根据税码自动计算。

因此程序只需传入金额和税码，SAP 会自动生成税额。这避免了接口端算税不一致的风险。

4. 调用 BAPI 创建预制发票

关键点

该 BAPI 不是直接过账发票，而是 Park Invoice（预制发票）。预制后的发票需要后续在 SAP 中审核过账。

返回值

返回值 说明
INVOICEDOCNUMBER 发票编号
FISCALYEAR 会计年度
RETURN 执行消息（内表）

业务流程总结

总结

本文梳理了使用 BAPI_INCOMINGINVOICE_PARK 创建预制采购发票的完整流程，核心职责可概括为：

  1. 根据采购订单及入库凭证生成发票行项目
  2. 计算发票总金额（含税）
  3. 组装发票抬头信息（凭证类型 RE、自动算税）
  4. 调用 BAPI_INCOMINGINVOICE_PARK 创建预制采购发票
  5. 创建成功后 COMMIT 事务并保存发票号到日志表
  6. 创建失败则 ROLLBACK 事务并返回错误信息

整个流程与上一篇文章《使用 BAPI_GOODSMVT_CREATE 进行采购入库（101）过账》配合，实现了：

采购订单 → 入库 → 采购发票（Park） 的完整 MM 业务闭环。

实际项目开发中建议将入库 + 发票创建封装为统一的接口 Function Module，统一处理事务提交、回滚和错误日志，避免在每个调用处重复编写判断逻辑。
