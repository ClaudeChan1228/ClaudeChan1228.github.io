使用 BAPI_GOODSMVT_CANCEL 冲销物料凭证

本文最后更新于 2026-06-26

引言

BAPI_GOODSMVT_CANCEL 是 SAP MM 模块中用于冲销物料凭证（Material Document） 的标准 BAPI。当入库、发货或其他物料移动过账后需要撤销时，调用该 BAPI 即可生成一张与原凭证数量相反的新冲销凭证，库存自动恢复。

本文基于实际接口开发经验，梳理从参数构造到事务提交流程的完整实现。

对应事务码：MBST（整张冲销）或 MIGO → Cancellation（按行冲销）。

整体流程

1. 定义变量

各变量说明：

变量 类型 作用
lt_return bapiret2 BAPI 返回消息（内表）
lt_item bapi2017_gm_item_04 需要冲销的行项目（内表）
ls_headret bapi2017_gm_head_ret 返回的新冲销凭证号
lv_mblnr mblnr 需要冲销的物料凭证号
lv_mjahr mjahr 物料凭证年度

2. 构造冲销行项目（可选）

goodsmvt_matdocitem 是一个可选参数。是否传入决定了冲销行为：

场景 传值 行为 对应事务码
整张冲销 空 冲销整张物料凭证 MBST
指定行冲销 有值 仅冲销指定行项目 MIGO → Cancellation

整张冲销（不传入行项目）

goodsmvt_matdocitem 为空 → 整张物料凭证全部冲销。

指定行项目冲销

仅冲销行 0002 和 0005，其他行保持不变。

从数据源动态构造

3. 调用 BAPI

主要参数

参数 方向 必填 说明
MATERIALDOCUMENT 传入 ✅ 需要冲销的物料凭证号
MATDOCUMENTYEAR 传入 ✅ 物料凭证年度
GOODSMVT_PSTNG_DATE 传入 冲销过账日期（可设为 sy-datum）
GOODSMVT_MATDOCITEM 传入 需要冲销的行项目（为空=整张冲销）
GOODSMVT_HEADRET 返回 新生成的冲销物料凭证号
RETURN 返回 BAPI 返回消息

4. 判断是否成功

检查 RETURN 中是否存在错误消息：

BAPI 返回消息类型速查

TYPE 说明 处理
S 成功（Success） 忽略
W 警告（Warning） 通常可忽略
I 信息（Information） 忽略
E 错误（Error） ❌ 失败
A 终止（Abort） ❌ 失败
X 异常（Exception） ❌ 失败

判断逻辑： 只要 RETURN 中出现 A、E、X 任意一类消息，即视为冲销失败。

5. 成功提交

⚠️ 重要： 必须显式调用 BAPI_TRANSACTION_COMMIT，否则冲销不会真正保存到数据库。设置 WAIT = 'X' 确保同步提交完成后再执行后续逻辑。

6. 失败回滚

作用：

  • 回滚整个 LUW（Logical Unit of Work）
  • 避免数据不一致

7. 获取冲销后的物料凭证

BAPI 成功执行后，GOODSMVT_HEADRET 返回新生成的冲销凭证信息：

字段 说明
MAT_DOC 新生成的冲销物料凭证号
DOC_YEAR 凭证年度

与 BAPI_GOODSMVT_CREATE 的关系

BAPI_GOODSMVT_CANCEL 和 BAPI_GOODSMVT_CREATE 共同构成了物料凭证的完整生命周期：

BAPI 方向 说明
BAPI_GOODSMVT_CREATE 创建 生成新的物料凭证

常见问题

Q1: 冲销后原凭证会被删除吗？

不会。冲销会生成一张新凭证，数量与原凭证相反。原凭证仍然存在于 MSEG / MKPF 表中。

Q2: 可以多次冲销同一张凭证吗？

一般情况下不能。一张凭证只能被完整冲销一次。但按行冲销时，不同行可以分批冲销。

Q3: 冲销凭证能再次冲销吗？

理论上可以再次冲销冲销凭证（冲销的冲销 = 还原），但在实际业务中通常通过 MBST 反冲。

Q4: goodsmvt_matdocitem 传入空和传入空行（lt_item 为空内表）效果一样吗？

一样。两种情况均视为整张冲销。

总结

BAPI_GOODSMVT_CANCEL 的使用总结如下：

要点 说明
作用 冲销已过账的物料凭证
goodsmvt_matdocitem 为空 整张冲销（等同于 MBST）
goodsmvt_matdocitem 有值 仅冲销指定行项目
RETURN 存在 A/X/E 冲销失败，执行 ROLLBACK
冲销成功 必须 COMMIT，从 GOODSMVT_HEADRET-MAT_DOC 获取新凭证号
原凭证 不删除，保留在系统中

核心代码流程：

  1. 构造行项目（可选）→ 决定整张冲销还是按行冲销
  2. 调用 BAPI_GOODSMVT_CANCEL
  3. 检查 RETURN → 失败则 ROLLBACK
  4. 成功则 COMMIT → 获取新冲销凭证号

实际项目开发中建议将该 BAPI 封装为独立的 Function Module 或 Method，统一处理错误逻辑、事务提交和日志记录。
