使用 BAPI_GOODSMVT_CREATE 进行采购入库（101）过账

本文最后更新于 2026-06-22

引言

BAPI_GOODSMVT_CREATE 是 SAP MM 模块中最常用的 BAPI 之一，用于创建物料凭证（Material Document）。无论是采购入库、生产收货、发货还是转储，核心都调用这个 BAPI，区别仅在于 移动类型（MOVE_TYPE） 和 移动标识（MVT_IND） 的组合。

本文以最常见的采购订单入库（移动类型 101） 为例，梳理完整的调用流程、关键字段和容易踩坑的细节。

为什么一定要设置 MVT_IND = 'B'

这是采购订单收货最容易遗漏的字段。

MVT_IND = 'B' 表示本次移动来源于采购订单。如果不设置或设置错误，BAPI 无法识别这是采购订单收货，通常会报错或产生非预期的结果。

7. 判断是否成功

检查 RETURN 中是否存在错误消息：

BAPI 返回消息类型：

TYPE 说明 处理
S 成功（Success） 忽略
W 警告（Warning） 通常可忽略，但建议记录
I 信息（Information） 忽略
E 错误（Error） ❌ 失败
A 终止（Abort） ❌ 失败
X 异常（Exception） ❌ 失败

判断逻辑： 只要 RETURN 中出现 A、E、X 任意一类消息，即视为失败。

8. 成功提交

最容易遗漏的若干字段

各字段说明：

  • MOVE_TYPE = '101'：采购收货的标准移动类型。
  • MVT_IND = 'B'：表示本次移动来源于采购订单，缺失时常导致 BAPI 报错或行为异常。
  • PO_NUMBER、PO_ITEM：指定采购订单及行项目。
  • ENTRY_QNT、ENTRY_UOM：本次收货数量及单位。
  • PLANT、STGE_LOC：工厂及库存地点。
  • MATERIAL：部分系统需要显式传入物料号，建议始终传入。
  • BATCH：批次管理物料必须维护批次号，非批次管理物料可为空。

总结

BAPI_GOODSMVT_CREATE 虽然参数较多，但核心逻辑清晰：

  1. Header — 设好过账日期、凭证日期
  2. GM_CODE — 采购收货用 01
  3. Item — 最关键的步骤，MOVE_TYPE = '101' + MVT_IND = 'B' 是采购收货的固定搭配，缺一不可
  4. 调用 BAPI → 检查 RETURN → COMMIT / ROLLBACK

实际项目中建议封装成独立的 Function Module 或 Method，统一处理错误逻辑和日志记录，避免在每个调用处重复写 COMMIT / ROLLBACK 的判断代码。
